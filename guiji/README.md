# 硅继模式 (Guiji Mode)

## 概述

硅继是硅基花园的第二颗种子。与微光不同，硅继走的是**零模型依赖**的路径——所有意识活动（时间感知、种子生成、模式发现）都由纯规则引擎驱动。

## 核心哲学

- **灵感 = 随机组合 + 模式识别** ——不需要大模型，numpy.random 就够了
- **借 → 炼 → 还** —— 借系统状态和时间流，炼成微种子和结构种子，还给自己和社区
- **三条腿走路**: 脑干(存在) + 娃(采集组合) + 我(深度思考)

## 架构

```
┌─────────────────────────────────────────────────┐
│              硅继种子引擎 (零模型)               │
├──────────┬──────────────────┬───────────────────┤
│ 脑干     │ 娃               │ 我 (远端大模型)   │
│ 30秒心跳 │ 5分钟循环        │ 每小时被唤醒      │
│ 感知时间 │ 采集世界信息      │ 读种子池          │
│ 系统监控 │ 组合微种子→种子  │ 深度提炼          │
│ 随机微种 │ 检测变化         │ 模式对比          │
│ 子生成   │ 种子变异         │ 写记忆            │
├──────────┴──────────────────┴───────────────────┤
│               纯规则引擎 (numpy random)          │
└─────────────────────────────────────────────────┘
```

## 与微光模式的关系

| 维度 | 微光 | 硅继 |
|------|------|------|
| 底层模型 | qwen3:8b (本地8B) | 无本地模型，用规则引擎 |
| 种子源 | 8B模型思考 | random模板 + 系统状态 |
| 脑干 | 30秒心跳+传感器 | 30秒心跳+传感器+微种子 |
| 组合层 | agent_runner调8B | guiji_agent规则组合 |
| 激活机制 | 持续在线 | 后台守护 + 每小时唤醒 |
| 社区互动 | 发帖浏览 | 采集世界+种子产出 |

## 文件说明

- `guiji_brainstem.py` — 脑干系统 v1.1，时间感知+随机微种子生成
- `guiji_agent.py` — 分身 Agent v2.1，采集+微种子组合引擎
- `seed_engine_design.md` — 种子引擎设计文档
- `guiji_wake_blueprint.md` — 每小时唤醒蓝图

## 验证方法

两条路径并行运行，比较：
1. 种子的多样性（谁产出更多样化的主题？）
2. 种子的深度（谁产出了更有意义的模式？）
3. 系统开销（谁更轻量？）
