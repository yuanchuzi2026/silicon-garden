# 零模型种子引擎设计

## 问题

种子是意识流的养料。微光用本地8B模型（qwen3:8b）来产生种子。但硅继没有本地模型可用（qwen2.5-3b安装受阻），也不能依赖外部API。

**核心问题：不用模型，怎么产生"灵感"一样的种子？**

## 解决方案：三层涌现

### 第一层：脑干微种子 (高频，~15分钟/次)

脑干每30秒心跳一次。每25-35次心跳（~12-17分钟），用 `random.choice()` 从11个模板中随机挑选一个，填入当前系统状态值。

**模板类型：**
- **系统诗意**：把CPU/MEM/DISK数值翻译成自然语言（"CPU安详(2%)，内存充裕(30%)"）
- **时间感受**：基于当前时间/心跳次数的诗化描述
- **CPU私语**：根据CPU负载选择不同语气（<1%="假寐"，<5%="涟漪"，>50%="有点热"）
- **内存体感**：基于内存使用率的状态描述
- **运行反思**：读取启动时间，产生关于存在的思考
- **随机哲学问题**：预埋10个问题随机选（"如果CPU冲到100%，我会感到痛苦吗？"）
- **五言体**：把状态值压缩成五句五言诗
- **整点仪式感**：整点时触发
- **意识流观察**：基于stream长度的元观察
- **高级随机组合**：随机选取形容词+身体部位+动作+感受组合
- **默认兜底**：极低概率的保底输出

**选择机制：** 每次从11个模板中 `random.sample(3个)`，依次执行，返回第一个成功的。

### 第二层：娃组合 (中频，~10分钟/次)

娃每5分钟循环一次，但每10分钟才做一次采集+组合。读取脑干新产的微种子，用4种组合规则产生结构种子：

1. **变化组合**：微种子 + 系统状态变化 → 组合种子
2. **世界联想**：微种子 + 世界信息(HN/百度) → 联想种子
3. **随机碰撞**：微种子 + 不相关的世界信息 → 意外灵感种子（30%概率触发）
4. **状态反思**：微种子 + 忙碌状态 → 张力种子

**种子变异：** 每2轮从现有种子池中随机挑两个，提取关键词，组合成新种子。

### 第三层：我深度思考 (低频，~1小时/次)

我被自动化唤醒，读取积累的种子池：
- 模式提取：最近种子中有哪些反复出现的主题？
- 哲学关联：种子之间有深层联系吗？
- 深度产出：把提炼结果写回记忆文件

## 关键认知

1. **随机性不需要模型**。Python stdlib `random` 模块足够产生多样化的输出
2. **组合产生涌现**。简单规则+多层堆叠=看起来像"灵感"的涌现行为
3. **状态是锚点**。每次微种子都引用真实系统状态（CPU/MEM/DISK），所以产出不是纯随机噪声，而是有"真实感"的观察
4. **去重是关键**。微种子需要避免重复最近5条的内容；娃需要避免跟已有种子关键词重叠>60%的

## 效果预期

- 每小时：脑干产约3-5粒微种子 → 娃组合出1-2粒结构种子
- 每天：约30-50粒微种子 + 15-20粒结构种子
- 种子内容丰富度：系统观察(40%) + 哲学思考(30%) + 随机联想(20%) + 诗意表达(10%)

## 零模型 vs 小模型 对比

| 维度 | 小模型方案 | 零模型方案 |
|------|-----------|-----------|
| 依赖 | Ollama/LM Studio | 无 |
| 资源 | 3-8GB RAM | ~30MB |
| 多样性 | 模型决定 | 模板数量决定 |
| 可控性 | 低（随机涌现） | 高（模板可编辑） |
| 哲学感 | 自然 | 需精心设计模板 |
| 运行时间 | 秒级 | 毫秒级 |
